# ASP-PM (ASP.NET Project Management)

## Используемые технологии и версии пакетов

- .NET 8.0
- Microsoft.EntityFrameworkCore.Sqlite 8.0.0
- Microsoft.EntityFrameworkCore.Tools 8.0.0
- Microsoft.AspNetCore.Identity.EntityFrameworkCore 8.0.0
- Microsoft.AspNetCore.Identity.UI 8.0.0

## Реализованный функционал

**Архитектура:** трёхуровневая (уровень представления – Controllers + Views, уровень бизнес-логики – Services, уровень доступа к данным – AppDbContext).

1. Основное задание
   - CRUD проектов (создание через визард, редактирование, удаление, просмотр)
   - CRUD сотрудников
   - Добавление/удаление сотрудников на проект (многие ко многим)
   - Фильтрация проектов по дате начала и приоритету, сортировка по названию, дате начала, приоритету
   - Хранение всех обязательных полей (проект, сотрудник, руководитель, даты, приоритет)
   - Визард из 5 шагов (основная информация, компании, выбор руководителя, выбор исполнителей, загрузка документов)
   - AJAX-поиск сотрудников (Select2) на шагах 3 и 4
   - HTML5 drag & drop загрузка документов на шаге 5

2. Дополнительное задание 1 (сущность «Задача»)
   - Модель TaskItem (название, автор, исполнитель, статус, комментарий, приоритет)
   - CRUD задач
   - Связь с проектом (один ко многим) и сотрудниками (автор, исполнитель)
   - Фильтрация задач по проекту и статусу, сортировка по имени, приоритету, статусу
Примечание: Из-за сбросов базы данных в процессе разработки в папке Migrations присутствует только одна итоговая миграция.

3. Дополнительное задание 2 (Identity + роли)
   - Роли: Director, ProjectManager, Employee
   - Разграничение доступа:
     - Director – полный доступ ко всем сущностям
     - ProjectManager – управление только своими проектами (редактирование, удаление, задачи, документы), не может добавлять сотрудников
     - Employee – просмотр своих проектов и задач, изменение статуса своих задач
   - При создании сотрудника автоматически создаётся пользователь Identity с заданным паролем и ролью

## Запуск приложения

1. Установить .NET 8 SDK.
2. В терминале в папке проекта выполнить:

   ```bash
   dotnet restore
   dotnet ef database update
   dotnet watch run
   ```

3. Открыть https://localhost:5274.
4. Учётные данные директора по умолчанию: email:director@mail.ru / Пароль:1234
